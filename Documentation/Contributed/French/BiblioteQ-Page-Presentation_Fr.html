    Bienvenue !

    BiblioteQ s'efforce d'être une suite professionnelle d'archivage, de catalogage et de gestion de bibliothèque, utilisant une interface Qt et offrant une connectivité à PostgreSQL et à SQLite. Les protocoles Open Library, SRU, et Z39.50 sont utilisés pour récupérer les données des livres, revues, et les magazines. Le logiciel est disponible pour tous les principaux systèmes d'exploitation et devrait être compatible avec tout système qui supporte Qt. BiblioteQ est distribué sous une licence BSD.

    Résumé de BiblioteQ

    • Technologie libre et open source.
    • Support de Qt 5 LTS et de Qt 6 LTS.
    • Fonctionnalité uniforme sur les différentes plateformes.
    • Architectures supportées : Alpha ; ARM ; PowerPC ; Sparc64 ; X86 ; X86_64.
    • OS supportés : Android ; FreeBSD ; Linux ; Mac ; OS/2 ; OpenBSD ; Windows.
    • Cibles d'installation pour MacOS et Windows : make install.
    • Windows portable.
    • Langues supportés : Allemand ; Anglais ; Arabe ; Espagnole ; Français ; Grecque ; Hongrois ; Polonais ; Portugais ; Russe ; Tchèque.
    • Documentation.
    • Catalogage de livres, dvd, littérature grise, revues, magazines, cd de musique, collections de photos et jeux vidéo.
    • Archivage des documents importants.
    • Affichages personnalisés.
    • Données personnalisées sur les articles, informations sur les prix et devises.
    • Des hyperliens intégrés pour des recherches localisées d'articles similaires.
    • Importation de livres à partir de fichiers CSV.
    • Exportation des vues vers des fichiers CSV.
    • Recherche d'images de première de couverture via Amazon et Open Library.
    • Images de couverture avec possibilité de glisser-déposer.
    • Navigateur de fichiers.
    • Fichiers joints aux éléments.
    • Réservation d'articles.
    • Listes des documents en retard, des documents demandés et des documents emrpuntés.
    • Historique facultatif des réservations d'articles pour les "patrons".
    • Prise en charge de plusieurs sites SRU et Z39.50.
    • Syntaxes des notices MARC21 et UNIMARC.
    • Analyse des balises MARC locales. Livres, revues, magazines.
    • Extraction de données via le protocole standard Z39.50 pour les livres, les journaux et les magazines.
    • Requêtes pour les articles sélectionnés via Open Library
    • Lecteur PDF via Poppler, si disponible.
    • Prise en charge de l'impression, y compris les aperçus.
    • Rôles d'administrateur.
    • Informations sur les membres.
    • Comptes PostgreSQL avec divers privilèges.
    • Des capacités de recherche riches, y compris des requêtes SQL personnalisées.
    • Politiques de sécurité au niveau des rows pour PostgreSQL.
    • Requêtes de bases de données transactionnelles.
    • Pagination réelle PostgreSQL et SQLite.
    • Supporte pour la demande d'articles non disponibles
